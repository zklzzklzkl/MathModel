# Problem Type Router

> 用于 `mm-problem-intake` 和 `mm-model-strategy`。  
> 目标不是给题目贴单一标签，而是识别混合结构，并把题型映射到可验证的模型路线。

## Inputs

- 题面原文或摘要。
- 附件与数据字段摘要。
- 每个 top-level question 的动词：评价、预测、优化、设计、解释、分类、推荐、检验等。
- 提交要求：论文、报告、策略、算法、图表、预测结果、排名或可执行方案。
- 已知约束：时间、空间、容量、预算、风险、政策、物理规律。

## Output Schema

`mm-model-strategy` 写入 `reports/MODEL_CANDIDATES.md` 时，题型路由结果必须包含：

```markdown
## Problem Type Routing

| Question | Primary Type | Secondary Types | Evidence From Prompt/Data | Confidence | Notes |
| --- | --- | --- | --- | --- | --- |

## Model Route Recommendation

| Question | Preferred Model | Baseline Model | Backup Model | Not Recommended | Validation | Recommended Figures |
| --- | --- | --- | --- | --- | --- | --- |
```

Confidence 使用 `high / medium / low`。混合题必须保留 secondary types，不得强行单标签。

## Type Taxonomy

### 综合评价类

信号：评价、排序、优选、风险等级、综合指数、指标体系。  
首选：熵权/组合赋权 TOPSIS、PCA/factor ranking、灰色关联。  
基准：等权线性评分。  
验证：权重敏感性、指标删除、排名相关。  
不推荐：无指标方向说明的 TOPSIS，无一致性检验的 AHP。

### 预测类

信号：预测、估计、趋势、未来、分类、风险概率。  
首选：可解释回归/时间序列/树模型。  
基准：均值、最近值、线性回归、季节朴素。  
验证：时间切分、滚动验证、误差分布、残差诊断。  
不推荐：小样本深度学习、泄漏未来信息的随机切分。

### 优化决策类

信号：最优、配置、调度、选址、路径、策略、约束。  
首选：线性/整数/非线性规划，多目标优化，动态规划。  
基准：贪心、均匀分配、规则方案。  
验证：可行性、约束利用率、目标值对比、敏感性。  
不推荐：只有算法名、无变量/目标/约束的遗传算法。

### 机理建模类

信号：机制、演化、传播、因果、动态系统、物理过程。  
首选：差分/微分方程、系统动力学、灰箱模型。  
基准：比例模型、线性响应、守恒关系。  
验证：参数来源、情景响应、拟合误差、敏感性。  
不推荐：只有流程图没有方程或可计算关系。

### 数据挖掘类

信号：模式识别、聚类、异常、画像、特征提取。  
首选：聚类、降维、异常检测、规则挖掘。  
基准：描述统计、阈值规则。  
验证：轮廓系数、稳定性、人工解释、业务可用性。  
不推荐：无法解释聚类含义的纯算法输出。

### 空间分布类

信号：区域、位置、栅格、地理、距离、空间相关。  
首选：空间插值、空间回归、热点分析、选址模型。  
基准：区域统计、距离加权。  
验证：空间留一、残差地图、尺度敏感性。  
不推荐：忽略空间自相关的普通回归。

### 网络图论类

信号：节点、边、路径、流量、连通、中心、网络风险。  
首选：中心性、最短路、最大流、社区发现、鲁棒性分析。  
基准：度、连通分量、简单路径。  
验证：节点移除、流量对比、稳定性。  
不推荐：没有边权定义的漂亮网络图。

### 排队服务类

信号：到达率、服务率、等待、拥堵、窗口、排队。  
首选：M/M/c、离散事件仿真、仿真优化。  
基准：平均到达/服务率计算。  
验证：等待时间分布、资源占用率、极端情景。  
不推荐：只算平均值而不讨论波动和峰值。

### 资源配置类

信号：人员、物资、预算、容量、覆盖、分配、公平。  
首选：整数规划、多目标决策、鲁棒配置。  
基准：按比例分配、贪心覆盖。  
验证：约束满足、边际收益、权重敏感性。  
不推荐：只给评分不落到可执行配置表。

### 风险评估类

信号：风险、灾害、可靠性、脆弱性、预警、概率。  
首选：风险指数、贝叶斯/逻辑回归、极值或蒙特卡洛。  
基准：阈值规则、历史频率。  
验证：混淆矩阵、召回率、置信区间、敏感性。  
不推荐：没有校准或阈值解释的风险等级。

### 仿真模拟类

信号：随机、场景、复杂系统、策略试验、难以解析。  
首选：Monte Carlo、离散事件、agent-based、系统动力学。  
基准：确定性场景。  
验证：重复次数、随机种子、置信带、极端场景。  
不推荐：只跑一次且无不确定性。

## Routing Rules

1. 先按题目动词识别任务，再用附件字段验证可实现性。
2. 如果题目是“预测后优化”“评价后配置”“机理后预测”，必须保留链式结构。
3. 首选模型必须有 baseline；没有 baseline 的高级模型不能直接作为主路线。
4. 每个不推荐模型必须写出不推荐原因，而不是简单列黑名单。
5. 每个推荐模型必须配验证方法和图表证据。
6. 低置信路由要写入 `WORKFLOW_STATE.md`，等待人工或后续数据审查确认。
